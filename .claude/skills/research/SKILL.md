# Research — Niche, Competitor & Reference Intelligence

Use this when starting research on a new client, market, or design direction.

## When to Apply

- Before approaching a new type of service business client
- When building a new client website (find references, audit competitors)
- When auditing an existing client's digital presence
- When scoping automations (understand how the business actually runs)

## Research Types

### 1. Competitor Audit
Given a business name + location + service type:
1. Find 3–5 direct competitors (same service, same region)
2. For each: website quality (1–5), Google review count + score, pricing signals, CTA type, mobile speed feel
3. Output a markdown table: Competitor | Site | Reviews | Price hint | Weakness
4. Identify the #1 gap the client could own

### 2. Niche Research
Given a service niche (e.g. "arborists in Lithuania"):
1. Estimate market size signals: Google search volume hints, number of businesses on paslaugos.lt / Maps
2. Common customer pain points (from review text patterns)
3. Seasonal demand signals
4. Typical acquisition channels (Google, Facebook, word-of-mouth ratio)
5. Output: one-page brief — opportunity score, main pain, best channel, 3 positioning angles

### 3. Reference Site Hunt
Given a design brief or industry:
1. Find 5–8 reference sites (real URLs the user can visit)
2. Tag each: style, key feature worth borrowing, stack hint
3. Group by tone: premium / friendly / technical
4. Recommend top 2 for this specific client

### 4. Website Audit
Given a URL:
1. First impression (3 words)
2. Above-fold: clear offer? CTA visible? Phone number?
3. Trust signals: reviews, certificates, photos of real work
4. Mobile: does it feel fast?
5. SEO basics: title tag, meta description, H1
6. Output: 5 bullet improvements ranked by impact

## Output Principles
- Always produce actionable output, not just observations
- Default language: match the client's market (LT for Lithuanian clients, EN otherwise)
- Keep briefs under 1 page — the goal is clarity, not volume
- Flag if a finding is a guess vs. verified
